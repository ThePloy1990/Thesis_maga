# Структура проекта

Этот документ описывает правильную структуру проекта и основные соглашения для поддержания порядка в коде.

## Основные пакеты

Все исходный код находится в директории `src`, разделенной на следующие пакеты:

- `src/data_collection/` - сбор и обработка данных (цены, новости, сентимент)
- `src/core/` - основная бизнес-логика (прогнозирование, оптимизация портфеля)
- `src/bot/` - компоненты ботов (Telegram, Streamlit интерфейс)
- `src/services/` - инфраструктурные сервисы (Celery, API, БД)

## Импорты

Всегда используйте абсолютные импорты от корня проекта:

```python
# Правильно
from src.data_collection import StockDataCollector
from src.core.portfolio_optimizer import optimize_portfolio
from src.services.models import User

# Неправильно
from data_collection import StockDataCollector
from ..core.portfolio_optimizer import optimize_portfolio
```

## Запуск скриптов

Скрипты для запуска компонентов системы находятся в директории `scripts/`:

- `scripts/run_bot.bat` - запуск Telegram бота
- `scripts/run_api.bat` - запуск API сервера
- `scripts/run_data_collectors.bat` - запуск сборщика данных
- `scripts/test_*.bat` - скрипты для тестирования

## Файлы данных

Все собранные данные и результаты сохраняются в директории `data/`:

- `data/stock_*.csv` - данные о ценах акций
- `data/crypto_*.csv` - данные о криптовалютах
- `data/sentiment_*.json` - результаты анализа сентимента
- `data/market_snapshot_*.json` - "снимки рынка"

## Модели машинного обучения

Модели машинного обучения хранятся в директории `models/`:

- `models/catboost_*.cbm` - модели CatBoost для прогнозирования

## Правила именования

- Используйте `snake_case` для имен файлов, модулей, функций и переменных
- Используйте `PascalCase` для имен классов
- Добавляйте префикс `test_` к файлам с тестами

## Важно!

При внесении изменений в структуру проекта:

1. Обновите файлы `__init__.py` для экспорта нужных функций/классов
2. Проверьте, что скрипты запуска указывают на правильные модули
3. Убедитесь, что импорты во всех файлах соответствуют новой структуре

## Замечания по файлам данных

Не храните большие файлы данных в репозитории. Вместо этого:

1. Добавьте расширения файлов данных (`.csv`, `.json` и т.д.) в `.gitignore`
2. Реализуйте механизм автоматической загрузки данных при первом запуске
3. Для тестирования используйте небольшие примеры данных
